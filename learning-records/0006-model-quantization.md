# 模型量化与硬件约束认知

用户通过实测理解了 GGUF 量化原理：原始 FP16 模型（16GB）→ Q4 量化（4.7GB），用精度换体积。亲身体验了硬件瓶颈：GTX 1050（2GB 显存）无法加载 8B 模型，Ollama 自动 fallback 到 CPU 推理，导致 8B 模型仅 2-5 tok/s。已安装 hermes3:8b（4.7GB）和 qwen2.5:3b（1.9GB），理解 8B vs 3B vs 1.5B 的选型逻辑。

**Evidence**: 用户实际运行 `ollama run hermes3:8b`，观察到回复速度很慢（CPU 推理），切换到 qwen2.5:3b 后速度明显提升。

**Implications**:
- 后续开发中，3B 模型作为日常开发测试首选（速度快），8B 模型用于需要高质量的最终验证
- 理解模型选型是"质量 vs 速度"的 trade-off，不同场景用不同模型
- 硬件限制意味着未来可能需要在云端或更强机器上部署 8B+ 模型
