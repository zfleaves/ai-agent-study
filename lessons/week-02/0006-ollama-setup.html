🔧 第 6 课：Ollama 安装 + 模型选型

动手 第 2 周 Day 1 约 45 分钟

本课目标：在 Windows 上安装 Ollama，理解 GGUF 量化概念，学会根据场景选择合适的模型大小和量化级别。

一、什么是 Ollama？

Ollama 是一个本地模型运行工具。它把复杂的模型下载、量化加载、推理服务打包成简单的命令行操作。

🔑 Ollama 三大核心能力
  1. 模型管理：ollama pull 下载模型，ollama list 查看已安装
  2. 本地推理：在你自己电脑上运行，数据不出本机，不用联网
  3. HTTP API：启动后在 localhost:11434 提供 REST API，任何语言都能调用

类比：前端工程师的直觉

Ollama 前端
ollama pull hermes3:8b npm install react
ollama run hermes3:8b node index.js
ollama list npm ls
localhost:11434 localhost:3000

二、GGUF 与量化：为什么模型有"大小"之分？

原始模型 vs 量化模型

原始的 LLM 模型参数用 FP16（16 位浮点数） 存储。一个 8B（80 亿参数）的模型：

8B 参数 × 2 字节(FP16) = 16 GB 显存/内存
—— 你的电脑根本跑不动！

量化（Quantization） 就是把高精度参数"压缩"成低精度，用精度换速度：

FP16（16位） → 16 GB   ← 原始精度，几乎不损失质量
  Q8（8位）  →  8 GB   ← 整数精度，质量损失极小
  Q6（6位）  →  6 GB   ← 质量损失很小，体积减半
  Q4（4位）  →  4 GB   ← 最常用，质量损失可接受
  Q3（3位）  →  3 GB   ← 体积更小，质量明显下降
  Q2（2位）  →  2 GB   ← 极限压缩，质量损失较大
🔑 关键认知

GGUF 是 Ollama 使用的模型文件格式，里面包含了量化后的模型权重。你看到的 hermes3:8b-q4_K_M 就是一个 GGUF 格式的 4 位量化模型。Q4_K_M 是当前性价比最高的量化级别——质量接近原版，体积只有 1/4。

三、模型选型：8B vs 3B vs 1.5B，怎么选？

模型大小 参数量 Q4 体积 内存需求 速度 质量 适用场景
8B 80 亿 ~5 GB 8-16 GB 🐢 慢 ⭐⭐⭐⭐⭐ 主力模型，复杂任务
3B 30 亿 ~2 GB 4-8 GB 🐇 中 ⭐⭐⭐ 开发测试，简单任务
1.5B 15 亿 ~1 GB 2-4 GB 🚀 快 ⭐ 快速原型，自动化测试
⚠️ Hermes 3.1 只有 8B 和 3B 版本

Nous Research 发布的 Hermes 3.1 主要版本是 8B 和 3B。如果电脑配置不够跑 8B，可以用 Qwen 2.5 作为替代（有 1.5B、3B、7B、14B 等各种尺寸）。

四、动手：安装 Ollama

📥 Step 1：下载安装
  1. 打开 https://ollama.com/download
  2. 下载 Windows 版（.exe 安装包）
  3. 双击安装，一路下一步，完成
  4. 安装完成后，Ollama 会在后台运行（右下角托盘图标 🦙）
✅ Step 2：验证安装

打开终端（PowerShell 或 CMD），输入：

ollama --version

如果看到版本号（如 ollama version is 0.x.x），说明安装成功。

📦 Step 3：拉取第一个模型

先拉取我们的主力模型 Hermes 3.1 8B：

ollama pull hermes3:8b

这会下载约 5GB 的文件，需要 5-15 分钟（取决于网速）。

下载完成后，再拉取一个小模型做对比：

ollama pull qwen2.5:3b
🧪 Step 4：测试对话

直接在终端里和模型聊天：

ollama run hermes3:8b

试试这几个 Prompt：

>>> 你好！用中文介绍一下你自己
>>> 帮我写一个 TypeScript 的冒泡排序函数
>>> What is the capital of France?（测试英文能力）
>>> /bye  （退出聊天）
📊 Step 5：对比模型速度

用同一个 Prompt 测试两个模型，感受速度差异：

ollama run hermes3:8b --verbose
>>> 用 TypeScript 写一个函数，计算斐波那契数列第 n 项

ollama run qwen2.5:3b --verbose
>>> 用 TypeScript 写一个函数，计算斐波那契数列第 n 项

--verbose 会显示每次推理的 token 生成速度（tok/s），记录这两个数字！

五、常用 Ollama 命令速查

ollama list              # 查看已安装的模型
ollama pull <model>      # 下载模型
ollama run <model>       # 运行模型（交互对话）
ollama rm <model>        # 删除模型
ollama show <model>      # 查看模型详细信息
ollama ps                 # 查看正在运行的模型
ollama serve              # 启动 Ollama 服务（默认已后台运行）

六、本课总结

七、自测题

  1. Ollama 的三个核心作用是什么？
  2. 什么是量化（Quantization）？Q4 和 Q8 有什么区别？
  3. 8B 参数、Q4 量化的模型大概需要多少内存？
  4. 为什么要有 8B/3B/1.5B 不同大小的模型？各适合什么场景？
  5. ollama run --verbose 中的 --verbose 有什么用？
点击展开参考答案
  1. ① 模型管理（下载/删除/查看）② 本地推理（数据不出本机）③ 提供 HTTP API（localhost:11434）
  2. 量化是把高精度参数（FP16）压缩成低精度（如 4 位/8 位整数），用精度换速度。Q8 质量损失极小但体积大（~8GB），Q4 体积只有 1/4（~4GB），质量损失可接受，是性价比最高的选择
  3. 约 8-16 GB 内存。8B 参数 × 0.5 字节(Q4) ≈ 4GB 模型 + 推理开销 ≈ 总共需要 8-16GB
  4. 8B：质量最好，适合复杂任务（Agent 推理、RAG 回答）；3B：平衡之选，适合开发测试；1.5B：速度最快，适合快速原型和自动化测试。选模型是"质量 vs 速度"的 trade-off
  5. --verbose 显示每次推理的详细性能数据，最重要的是 token 生成速度（tok/s）——用来对比不同模型的推理速度
