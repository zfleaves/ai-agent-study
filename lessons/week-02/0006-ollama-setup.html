一、知识：Ollama 是什么

Ollama 是本地模型运行工具，把模型下载、量化加载、推理服务打包成命令行操作。三个核心能力：

  1. 模型管理：ollama pull 下载，ollama list 查看
  2. 本地推理：数据不出本机，完全离线可用
  3. HTTP API：localhost:11434 提供 REST API，任何语言调用
🔑 前端直觉
Ollama类比
ollama pull hermes3:8bnpm install react
ollama run hermes3:8bnode index.js
ollama listnpm ls

二、知识：GGUF 与量化

原始 LLM 参数用 FP16（16 位浮点数） 存储。8B 模型：

8B 参数 × 2 字节(FP16) = 16 GB — 你的电脑跑不动

量化（Quantization） 把高精度参数压缩成低精度，用精度换体积：

量化级别体积质量评价
FP1616 GB原始个人电脑跑不动
Q88 GB损失极小需要大内存
Q4_K_M~5 GB可接受⭐ 性价比最高
Q3~3 GB明显下降极限压缩
Q2~2 GB损失较大慎用
🔑 GGUF

GGUF（GPT-Generated Unified Format）是 Ollama 使用的模型文件格式，包含量化后的权重。你看到的 hermes3:8b 默认就是 Q4_K_M 量化——体积 1/4，质量损失可接受。

三、知识：模型选型

大小参数量Q4 体积内存需求速度质量场景
8B80 亿~5 GB8-16 GB🐢⭐⭐⭐⭐⭐主力模型
3B30 亿~2 GB4-8 GB🐇⭐⭐⭐开发测试
1.5B15 亿~1 GB2-4 GB🚀⭐快速原型
⚠️ 你的硬件

GTX 1050 只有 2GB 显存，8B 模型（4.7GB）塞不进 GPU，Ollama 自动 fallback 到 CPU 推理。日常开发建议用 qwen2.5:3b（1.9GB），速度快 3-5 倍。Hermes 目前只有 8B/3B 版本，3B 质量一般，日常用 qwen2.5:3b 替代。

四、动手

📥 Step 1：安装
  1. 打开 ollama.com/download
  2. 下载 Windows 版，双击安装，一路下一步

五、命令速查

ollama list           # 已安装的模型
ollama pull <model>   # 下载模型
ollama run <model>    # 运行模型（交互对话）
ollama rm <model>     # 删除模型
ollama ps              # 正在运行的模型
ollama serve           # 启动 API 服务

六、自测

用纸笔回答，不看答案。

  1. Ollama 的三个核心作用？（各 5 字）
  2. 量化是什么？Q4 比 FP16 省多少？
  3. 8B Q4 模型大约需要多少内存？
  4. 你电脑上为什么 8B 比 3B 慢很多？
  5. --verbose 显示的最关键指标是什么？
参考答案
  1. 模型管理（下载/删除/查看模型）、本地推理（数据不出本机，完全离线可用）、HTTP API（在 localhost:11434 提供 REST API，任何语言都能调用）。类比：模型版的 Docker。
  2. 量化是把高精度参数（FP16，16位浮点数）压缩成低精度整数（如 4 位），用精度换体积。Q4 体积约是 FP16 的 1/4（16GB → 4GB），质量损失可接受，是性价比最高的量化级别。GGUF 是 Ollama 使用的量化模型文件格式。
  3. 约 8-16 GB 内存。8B 参数 × 0.5 字节(Q4) ≈ 4GB 模型文件本身，但推理时还需要额外内存做中间计算，实际占用 8-16GB。如果内存不够，系统会用磁盘 swap，速度会进一步下降。
  4. 因为 GTX 1050 只有 2GB 显存，而 8B 模型 Q4 量化后约 4.7GB，根本塞不进显存。Ollama 自动 fallback 到 CPU 推理——CPU 算力远不如 GPU，所以 8B 模型在 CPU 上只有 2-5 tok/s。3B 模型（1.9GB）虽然也跑在 CPU 上，但计算量小得多，速度自然快 3-5 倍。
  5. eval rate（tokens/s），即模型每秒能生成多少个 token。这是衡量推理速度的最核心指标，直接决定了你看到回复"蹦字"的快慢。load_duration 和 prompt_eval_duration 也很重要，但 eval rate 是用户最直接感知到的。
