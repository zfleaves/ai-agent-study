二、自测题（5 题）

第 1 题

❓ LLM 和 Agent 有什么区别？用一句话解释。
点击展开答案

LLM 是"脑子"（推理能力），Agent 是 LLM + 工具 = 有"手"的 LLM。LLM 只能"说"，Agent 能"做"——查天气、搜网页、读文件、执行代码。本质区别：Agent 能调用外部工具，与真实世界交互。

第 2 题

❓ 什么是 Token？Context Window 是什么？为什么 Context Window 很重要？
点击展开答案

Token：模型读写文本的最小单位。中文 1 字 ≈ 1-2 token，英文 1 词 ≈ 1-2 token。
Context Window：模型一次能"看到"的最大 token 数（如 Hermes 3 8B 有 8192 token 窗口）。
为什么重要：它决定了对话记忆的上限。超过窗口的对话会被"遗忘"，所以需要记忆管理和历史截断策略。

第 3 题

❓ ReAct 循环中的 Thought、Action、Observation 分别是什么？循环什么时候终止？
点击展开答案

Thought：Agent 的内部推理——"用户想要什么？我需要什么信息？"
Action：调用工具（查天气、搜网页）或输出最终回答
Observation：工具返回的结果，反馈给 Agent 继续推理

终止条件：① 正常终止——Agent 判断信息够了，生成最终回答；② 强制终止——达到 maxSteps 上限，防止无限循环。

第 4 题

❓ AI Agent 技术栈分为哪四层？MCP 协议在哪一层？它解决什么问题？
点击展开答案

四层（从下到上）：
① 🏗️ 模型层（Ollama、vLLM）—— 运行模型
② 🔧 工具&数据层（MCP、LanceDB、SQLite）—— 工具协议、向量存储、持久化
③ 🧠 框架层（Vercel AI SDK、LangChain.js）—— 编排 Agent 逻辑
④ 🎨 应用层（Next.js、React）—— 用户界面

MCP 在工具&数据层。它解决的是工具与 Agent 的耦合问题——一套标准协议，让任何 Agent 都能连接任何工具，就像 USB-C 一样。
