📖 第 1 课：Hermes Agent 是什么？

概览 入门 约 10 分钟

本课目标：建立 Hermes Agent 的完整认知地图。知道"有哪些东西"、"它们之间的关系"、"我应该关注哪个"。

先说核心问题：Hermes Agent 到底指什么？

当你搜索 "Hermes Agent" 时，会撞上两个不同的东西，它们名字相同但本质不同：

🧠 Nous Hermes 模型

开源 LLM

由 Nous Research 团队开发的一系列开源大语言模型。基于 Llama、Mistral 等基础模型，通过特殊训练让模型学会使用工具。

类比：一个被训练成"会使用计算器、搜索引擎、API"的 AI 大脑。

📦 产物：模型权重文件（可下载、本地运行）

🔬 HERMES 学术框架

学术论文

一篇提出 Web Agent 自学习框架的学术论文。核心思想是让 Agent 在浏览网页时，记住成功经验，下次遇到类似任务能直接复用。

类比：一个"做网页任务时会记笔记、下次翻笔记"的智能助手。

📦 产物：论文 + 算法设计（不一定有成熟开源实现）

💡 关键区分

Nous Hermes = 你可以下载运行的模型（产品/工具）
HERMES 论文 = 一种 Agent 学习方法的理论设计（思想/框架）

两者都涉及"Agent 学习"，但一个在训练阶段学习（模型训练时），一个在运行阶段学习（Agent 执行任务时）。

分支一：Nous Hermes 模型系列

它是什么？

Nous Research 是一个开源 AI 研究团队。他们的 Hermes 系列是目前最流行的开源 Agent 模型之一。核心理念：

"让开源模型像 GPT-4 一样能调用工具、执行多步任务。"

版本演进

版本 基础模型 核心改进
Hermes 2 Llama 2 / Mistral 首次引入 tool-calling 能力
Hermes 2 Pro Mistral 增强 function calling 精确度
Hermes 3 Llama 3.1 128k 上下文 + 更强的多步推理
Hermes 3.1 Llama 3.1 GRPO 偏好对齐，Agent 轨迹优化

它是怎么"学习"的？

Hermes 模型的"学习"发生在训练阶段，核心流程：

  1. 合成数据生成 — 用 GPT-4/Claude 生成大量"任务 + 工具调用"的对话数据
  2. 监督微调（SFT） — 在这些数据上训练模型，让它学会"看到问题 → 选择合适的工具 → 解析结果"
  3. 偏好对齐（DPO/GRPO） — 让模型偏好"能成功完成任务的"行动序列，而不是"看起来合理但会失败的"
  4. 迭代改进 — 用模型自己的成功/失败轨迹继续训练，逐步提升
⚠️ 新手常见误区

Hermes 模型不会在和你对话时继续学习。它的"学习"全部发生在训练阶段。部署后，模型的权重是冻结的——它不会因为你的一次对话就变聪明。

（对话历史的记忆 ≠ 学习，那只是把历史塞进上下文窗口。）

分支二：HERMES 学术框架

它是什么？

全称通常是 "HERMES: A Retrieval-Augmented and Self-Learning Framework for Web Agents"（或类似变体），是一个让 Web Agent 在执行任务时不断自我改进的框架设计。

核心思想

人类遇到新任务时：先回想以前有没有做过类似的事 → 有的话参考经验 → 没有的话尝试 → 记录结果下次用

HERMES Agent 也是：检索历史经验（RAG）→ 模仿成功轨迹（行为克隆）→ 探索新方案（强化学习）→ 记录结果

它是怎么"学习"的？

  1. 经验库（RAG） — 维护一个"历史成功轨迹"数据库，遇到新任务先检索最相关的经验
  2. 行为克隆（冷启动） — 先用人类示范数据学会基本操作
  3. 强化学习（在线提升） — 在真实环境中尝试，成功就强化，失败就调整
  4. Critic 模型 — 一个独立的"评分员"模型，对每个子步骤打分，提供密集反馈
⚠️ 注意

这个框架目前主要在学术阶段，不一定有成熟的开源实现可以直接用。它的价值更多在于提供了一种"Agent 应该如何自我改进"的设计思路。
