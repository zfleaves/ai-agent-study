记住这个四层结构，后面每学一个新技术，你都能把它"放进"对应那一层。

三、逐层详解

🏗️ 第 1 层：模型层（Model Layer）

模型层是 Agent 的"大脑"。负责：加载模型、运行推理、返回 token。

技术 定位 我们用它做什么 类似前端中的…
Ollama 本地模型运行工具 一键下载、运行、管理本地模型（Hermes、Qwen 等） Node.js 运行时
vLLM 高性能推理引擎 生产环境高并发推理（比 Ollama 快 10x+），第 15 周学 Nginx 反向代理
HuggingFace 模型仓库 + 社区 浏览模型卡、下载 GGUF 量化模型、看社区评测 npm registry
OpenAI API 云端模型 API 作为云端兜底方案（本地模型跑不动时用云端） CDN / 云服务
🔑 关键认知

Ollama 是我们第 2-10 周的主力模型层工具。它就像 Docker 之于容器——把复杂的模型下载、量化、运行全部简化成几条命令。vLLM 是生产级方案，第 15 周学。

🧠 第 2 层：框架层（Framework Layer）

框架层是 Agent 的"指挥系统"。负责：编排工具调用、管理 Agent 循环、处理流式响应、连接前端。

技术 定位 核心价值 学习周
Vercel AI SDK 前端原生的 AI 框架 React Hook 直接连模型，Streaming 开箱即用，TypeScript 一等公民 第 7 周
LangChain.js 通用 AI 应用框架 工具链最全（RAG、Agent、Chain），生态最大，但抽象层较重 第 5 周起接触
LangGraph 状态图编排引擎 把 Agent 流程建模为有向图（节点+边），支持条件分支、循环、人工审批 第 10 周
CrewAI 多 Agent 协作框架 定义角色、分配任务、Agent 间通信，Python 原生（TS 版较弱） 第 10 周了解
⚠️ 框架选择的坑

很多新手一上来就学 LangChain，结果被它复杂的抽象绕晕。我们的策略：先手写、再用框架。第 3-4 周手写 Agent 循环，第 5-6 周手写 RAG 管道，第 7 周才引入 Vercel AI SDK。等你能手写之后再学框架，你会觉得"原来框架就是帮我省了这些代码"——而不是"我不知道框架在干什么"。

🔧 第 3 层：工具 & 数据层（Tooling & Data Layer）

这一层是 Agent 的"工具箱"和"记忆库"。负责：存储知识、检索信息、持久化状态、标准化工具协议。

技术 定位 解决什么问题
MCP Model Context Protocol 工具调用的"USB-C 接口"——标准化 Agent 和工具的连接方式，一套协议对接所有工具
LanceDB 向量数据库（TS 原生） 存储文本的向量表示，支持语义搜索（"意思相近"而非"关键词匹配"）
Chroma 向量数据库（Python 原生） 与 LanceDB 同类，Python 生态更成熟，第 13 周 Python 阶段使用
SQLite 轻量关系型数据库 存会话历史、用户偏好等结构化数据，零配置，文件即数据库
Redis 内存缓存数据库 生产环境替代 SQLite 做会话缓存，TTL 自动过期，高并发支持
🔑 关键认知：MCP 是 2026 年最重要的 Agent 标准

Anthropic 在 2024 年底提出的 MCP 协议，正在成为 Agent 工具集成的行业标准。它的核心思想是：工具提供方只需实现一个 MCP Server，任何 Agent 框架都能通过 MCP Client 连接它。类比：MCP 之于 Agent 工具，就像 HTTP 之于 Web 服务——一套标准协议，解耦提供方和消费方。

🎨 第 4 层：应用层（Application Layer）

应用层是用户看到的界面。对我们来说，就是前端技术栈。

技术 在 Agent 应用中的角色
Next.js / React 构建 Agent 对话界面、知识库管理后台
Vercel AI SDK（前端部分） useChat hook 处理 Streaming、工具调用 UI 状态、消息管理
Tailwind CSS 快速构建美观的 Chat UI（你熟悉，无需额外学）
✅ 前端工程师的优势

这一层你已经是专家了。AI Agent 开发中，能把 Agent 逻辑和 UI 都做好的人是稀缺的——大多数 AI 开发者只会 Python 后端，不会做 UI。你的前端背景是 差异化竞争力。

🔑 为什么模型层三语言都一样？

Ollama 是一个独立服务（HTTP API），不管你的代码用 TS、Java 还是 Python 写的，都是通过 HTTP 调用它。这就是服务解耦的好处——模型层和框架层互不绑定。

八、常见困惑解答

❓ "LangChain.js 和 Vercel AI SDK 有什么区别？我该学哪个？"

Vercel AI SDK：轻量级，前端友好，Streaming 处理开箱即用。useChat hook 直接拿去做 UI，适合快速构建 Agent 对话应用。LangChain.js：全功能框架，生态最全（RAG、Agent、Chain、Tool 等模块都有），但抽象层重，学习曲线陡，适合复杂 AI 管道。
我们的策略：两个都学，先 AI SDK 后 LangChain。快速做产品用 AI SDK，需要完整工具链用 LangChain。先手写理解原理，再用框架提升效率。

❓ "MCP 协议和直接写 Tool 函数有什么区别？"

直接写 Tool 函数：你写一个 TS 函数，Agent 直接调用它。适合简单场景。
MCP 协议：工具以独立 Server 形式运行，Agent 通过标准协议发现和调用。适合：① 工具需要复用（多个 Agent 共用）② 工具由不同团队维护 ③ 想用社区现成的 MCP Server。我们第 8 周会详细对比。

❓ "Ollama 和 vLLM 有什么区别？为什么先学 Ollama？"

Ollama：本地开发友好，一条命令启动模型，适合学习和原型。核心作用就三个：① 模型管理（ollama pull，类比 npm install）② 本地推理（数据不出本机）③ 暴露 HTTP API（localhost:11434）。类比：Ollama = Node.js 运行时 + npm registry 合体。
vLLM：生产级推理引擎，高并发、高吞吐，但配置复杂。
类比：Ollama 是 npm run dev，vLLM 是 nginx + PM2 集群部署。先学会开发，再学部署优化。

点击展开参考答案
  1. 四层架构（从下到上）：
    🏗️ 模型层：Ollama（本地运行模型）
    🔧 工具&数据层：MCP（工具标准协议）、LanceDB（向量数据库）、SQLite/Redis（持久化）
    🧠 框架层：Vercel AI SDK（轻量，前端友好）、LangChain.js（全功能，生态大）
    🎨 应用层：Next.js + React + Tailwind CSS（用户界面）
  2. Ollama 属于模型层，有三个核心作用：
    ① 模型管理：一条命令下载模型（ollama pull hermes3:8b），类比 npm install
    ② 本地推理：在你电脑上运行模型，数据不出本机，不用联网
    ③ 暴露 HTTP API：启动后在 localhost:11434 提供 REST API，TS/Java/Python 代码通过 HTTP 调用
    类比：Ollama = Node.js 运行时 + npm registry 合体，既能"安装"模型，又能"运行"模型，还能被外部调用。
  3. MCP 解决的是工具与 Agent 的耦合问题（不是"工具问题"本身）。
    没有 MCP：每个 Agent 框架自定义工具格式，同一个天气工具要写两套适配代码 → 工具无法复用
    有了 MCP：工具只需实现一次 MCP Server，任何实现了 MCP Client 的 Agent 都能连接它
    就像 USB-C——充电器、耳机、显示器都能插同一个口，不用每个设备配一根专用线。一套协议对接所有工具。
  4. Vercel AI SDK：适合前端驱动的 Agent 对话应用。React 聊天界面 + Streaming + 工具调用可视化，useChat hook 直接拿去做 UI，开箱即用。
    LangChain.js：适合复杂 AI 管道。RAG（文档→分割→向量化→检索→生成）一条龙，模块全但抽象层重，学习曲线陡。
    一句话：快速做产品用 AI SDK，需要完整工具链用 LangChain。我们课程两个都学，先 AI SDK（第 7 周），后 LangChain（贯穿使用）。
  5. Node.js + TypeScript + Ollama + Hermes 3.1 模型，就 4 个。前两周命令行交互，暂时不需要前端框架。
