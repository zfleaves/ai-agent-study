基础概念

LLM（Large Language Model，大语言模型）
在海量文本上训练的神经网络模型，能理解和生成自然语言。GPT-4、Claude、Llama 都是 LLM。Hermes 是基于 Llama 微调的 LLM。
Agent（智能体）
具备自主决策和行动能力的 AI 系统。与普通聊天机器人不同，Agent 能主动调用工具、执行多步计划、根据环境反馈调整策略。类比：聊天机器人是"图书管理员"，Agent 是"侦探"。
Tool-calling / Function Calling（工具调用）
让 LLM 输出结构化的函数调用请求，而不是纯文本。例如：模型不直接回答"现在几点"，而是输出 get_time() 这个函数调用，由外部程序执行后把结果返回给模型。
Token
LLM 处理文本的最小单位。英文中约 1 个词 ≈ 1.3 个 token，中文中约 1 个字 ≈ 1.5-2 个 token。模型的上下文窗口大小以 token 计（如 128k = 约 10 万中文字）。
Context Window（上下文窗口）
模型一次最多能"看到"的 token 总量。超过这个限制，最早的内容会被截断。Hermes 3 支持 128k 上下文窗口。

训练方法

SFT（Supervised Fine-Tuning，监督微调）
在标注好的数据上训练模型。给模型看"问题 + 正确答案"，让它学会模仿。Hermes 的 SFT 数据是 GPT-4 生成的合成对话。类比：学生做有答案的练习题。
Synthetic Data（合成数据）
用更强的 AI 模型（如 GPT-4、Claude）生成训练数据，而不是人工标注。优点是量大、成本低、质量可控。Hermes 的核心训练数据就是合成数据。
RLHF（Reinforcement Learning from Human Feedback，人类反馈强化学习）
让人工标注员对模型输出打分，用这些分数训练一个"奖励模型"，再用强化学习让 LLM 最大化奖励。问题是：人工标注贵、慢、不一致。
DPO（Direct Preference Optimization，直接偏好优化）
RLHF 的替代方案。不需要单独训练奖励模型，直接在"好答案 vs 坏答案"的对比数据上优化。更简单、更稳定、更便宜。Hermes 3+ 的核心对齐方法。
GRPO（Group Relative Policy Optimization，组相对策略优化）
DPO 的进阶版。一次生成一组回答，组内相对比较，好的强化、差的抑制。比 DPO 更稳定，不需要外部参考模型。
Preference Alignment（偏好对齐）
让模型"偏好"某种输出风格或行为模式的训练过程。不只是"答对"，而是"答得好"——遵循指令、安全、有用。DPO 和 RLHF 都是对齐方法。

Agent 学习机制

Agentic Trajectory（Agent 轨迹）
Agent 执行一次任务从头到尾的完整记录：每一步的观察 → 思考 → 工具调用 → 结果。用于训练 Agent 模型。类比：棋谱记录了每一步的走法。
Behavioral Cloning（行为克隆）
从人类示范中直接学习"怎么做"。收集人类完成任务的操作记录，让模型模仿。优点是快速冷启动，缺点是遇到没见过的情况容易出错。
Self-Play（自对弈）
模型自己和自己对抗/交互来产生训练数据。AlphaGo 用自对弈变强，Agent 也可以用自对弈探索更多任务场景。
RAG（Retrieval-Augmented Generation，检索增强生成）
在生成回答前，先从知识库中检索相关信息，再结合检索结果生成回答。HERMES 学术框架用 RAG 检索历史成功经验。
Critic Model（评判模型）
一个独立的模型，专门用来评估 Agent 的每个动作"好不好"。提供密集的、细粒度的反馈信号，而不是只在任务完成时给一个总评。

模型与部署

Ollama
在本地电脑上运行开源 LLM 的轻量工具。一条命令下载并运行模型，支持 Windows/Mac/Linux。部署 Hermes 模型的首选方式。
HuggingFace
最大的开源 AI 模型托管平台。可以在这里找到 Hermes 模型权重、阅读模型卡、下载数据集。GitHub 的 AI 版。
Weights（模型权重）
神经网络中每个连接的参数值。模型文件（如 .gguf、.safetensors）本质上是大量权重数字的集合。"下载模型" = 下载这些权重文件。
Quantization（量化）
压缩模型权重以降低显存/内存占用和加速推理。例如 4-bit 量化可以让 8B 模型在普通电脑上运行。代价是轻微的质量损失。
