🛠️ 环境准备清单

开始学习前必读 建议提前准备

本清单覆盖 12 周学习所需的所有软件、硬件、账号和前置知识。建议在开始 Day 2 之前全部就绪。

1. 硬件要求

项目 最低配置 推荐配置 说明
内存 16 GB 32 GB 运行 8B 模型（Q4 量化）需要 ~6GB，加上系统开销，16GB 是最低门槛
硬盘 30 GB 空闲 50 GB+ Hermes 3.1 8B ~4.7GB，Embedding 模型 ~1GB，其他模型和项目代码
GPU 不需要 NVIDIA 8GB VRAM+ Ollama 支持纯 CPU 运行，有 GPU 会更快，但不是必须
操作系统 Windows 10+ Windows 11 Ollama 原生支持 Windows，无需 WSL
⚠️ 内存不足怎么办？

如果你的电脑只有 8GB 内存，可以改用更小的模型：

  • Hermes 3.1 3B（Q4 ~1.8GB，功能完整但能力稍弱）
  • Qwen2.5 1.5B（Q4 ~1GB，中文好，极轻量）
  • 学习内容不变，只是模型小一些，回答质量有差距

2. 软件安装清单

2.1 必须安装

软件 版本 用途 安装方式
Node.js 18.x 或 20.x 运行 TypeScript 项目 nodejs.org 下载 LTS 版
Ollama 最新版 本地运行 LLM 模型 ollama.com 下载 Windows 版
Git 2.x 版本控制 git-scm.com
VS Code 最新版 代码编辑器 code.visualstudio.com

2.2 推荐安装

工具 用途
Windows Terminal 更好的终端体验（多标签、UTF-8、主题）
Postman 或 Bruno 测试 Ollama API（之后会用到）
Docker Desktop 第 12 周部署时使用（可以到时候再装）

2.3 VS Code 推荐扩展

  • ESLint — 代码规范
  • Prettier — 代码格式化
  • GitHub Copilot（可选）— AI 辅助编码

3. 安装验证步骤

✅ 安装完成后，逐条验证

3.1 验证 Node.js

node --version
# 应输出: v18.x.x 或 v20.x.x

npm --version
# 应输出: 9.x.x 或 10.x.x

3.2 验证 Git

git --version
# 应输出: git version 2.x.x

3.3 验证 Ollama

ollama --version
# 应输出: ollama version x.x.x

ollama list
# 应输出: NAME  ID  MODIFIED  SIZE
# （此时应该为空，还没有下载模型）

3.4 验证 TypeScript

npx tsc --version
# 应输出: Version 5.x.x

4. 模型下载清单

这些模型会在对应周下载，但可以提前了解：

模型 大小 用途 下载命令 使用周
Hermes 3.1 8B ~4.7 GB 主力 Agent 模型 ollama pull hermes3:8b 第 2 周起
nomic-embed-text ~274 MB 文本向量化（Embedding） ollama pull nomic-embed-text 第 5 周起
Qwen2.5 7B ~4.4 GB 中文场景备选模型 ollama pull qwen2.5:7b 第 6 周（可选）
⚠️ 下载注意
  • 模型文件较大，建议在网络好的时候提前下载
  • 下载后模型存储在 %USERPROFILE%\.ollama\models\
  • 如果 C 盘空间不够，可以修改 Ollama 模型存储路径

8. 准备时间估算

步骤 预计耗时 备注
安装 Node.js 5 分钟 如果已安装，验证版本即可
安装 Ollama 5 分钟 下载安装包 + 安装
下载 Hermes 3.1 8B 10-30 分钟 取决于网速，约 4.7GB
下载 Embedding 模型 2-5 分钟 约 274MB，很快
初始化项目 5 分钟 npm init + tsconfig
总计 约 30-60 分钟 大部分时间在等模型下载
💡 建议

模型下载可以后台挂着——下载的同时就开始 Day 2 的理论学习，互不耽误。下载命令：

ollama pull hermes3:8b

这条命令执行后，模型就开始下载，你可以在另一个终端继续学习。
